1. WHAT IS OCHESTRATION IN BIG DATA?
Ochestration is simply coordinating multiple data tasks so they run in the correct order automatically.

Example pipeline:
Extract data  -> Clean data → Train model → Send report
without orchestraction:
❌ You run everything manually.
With orchestration:
✔ System runs automatically in order.

2. NEED FOR DEPENDENCY MANAGEMENTIN DATA PIPELINES
PROBLEM WITH DEPENDENCIES
IMAGINE:
task b DEPENDDS ON task a
if B runs first ... it leads to failureand wrong data

Dependenmcy management means: you define "Task B should only run after task A succceeds"

4. Airflow Architecture and components 
MAIN COMPONENTS:
*) DAG (DIRECTED ACYCLIC GRAPH)
    A DAG =  your pipeline definition
    this describes: tasks, and order of execution
*) Scheduler: 
    This is the brain
    it decides:
    . when tasks should run 
    . what runs next
*) Worker (Executor)
    Actual worker that runs tasks.
*) Metadata Database:
    Stores: 
    . task status
    . logs
    . execution history
*) Web Ui:
    Dashboard where you see:
    . task progress
    . failures
    . retries

ARCHITECTURE FLOW
DAG file → Scheduler → Worker → Database → UI\
================
